🔀 What is Control Flow?

By default, JavaScript runs every line from top to bottom — one after another. Control flow lets us break that pattern. We can skip over code, or only run it when a condition is true.

💡 Real-world example: "If the user has entered their password correctly, log them in. Otherwise, show an error." That decision is control flow.

🗺️ How If / Else Works

Think of it as a fork in the road:

if (condition)
↓
TRUE ✅
↓
Run Block A
FALSE ❌
↓
Run Block B (else)
↓
Continue program…

🔁 Why Do We Need Loops?

Imagine printing every number from 1 to 100. Without a loop, you'd write 100 lines of code. With a loop, you write 3. Loops let you repeat a block of code efficiently.

📌 Use a for loop when you know exactly how many times you need to repeat something.

🔬 Anatomy of a For Loop

Every for loop has three parts separated by semicolons:

① Initialization: let i = 1
↓
② Condition: i <= 5 ?
TRUE
↓
③ Body runs
↓
④ i++ (increment)
↑ back to ②

🔄 The While Loop

A while loop keeps running as long as a condition is true. Unlike a for loop, you don't specify a fixed number of iterations up front — it runs until the condition changes.

📌 Use a while loop when you don't know how many times the loop needs to run — for example, "keep asking for input until the user enters a valid number." 
while (condition) {
    // Body runs as long as condition is true
    // ⭐ Make sure something inside changes the condition!
}

⚖️ For vs While — When to Use Which

SituationUse
You know exactly how many iterationsfor
You're counting up or down through a rangefor
You don't know how many iterations in advancewhile
Waiting for a user action or external eventwhile
Running until a specific value is reachedwhile
💡 Analogy: A for loop is like a dishwasher set to run 3 cycles. A while loop is like washing dishes by hand — you keep going until they're all clean, however many cycles that takes.

📖 What is a Function?

A function is a named, reusable block of code that performs a specific task. You define it once, and you can call (invoke) it as many times as you need.

💡 Analogy: A function is like a recipe in a cookbook. Writing the recipe doesn't cook the food — you have to actively decide to cook it (calling the function). And the same recipe can produce 10 different meals (calling the function with different arguments).

🔬 Function Anatomy

// ↓ keyword  ↓ name      ↓ parameters
function   greetUser  (firstName, lastName) {
    // ← the function body
    console.log(`Welcome, ${firstName} ${lastName}!`);
}

// Calling the function with arguments:
greetUser("Alice", "Johnson"); // → Welcome, Alice Johnson!
greetUser("Bob",   "Smith");   // → Welcome, Bob Smith!

↩️ The Return Statement — Most Important Concept!

There's a critical difference between a function that shows you something and one that gives you something back.

❌ console.log inside (data is lost)
function addWrong(a, b) {
  console.log(a + b); // shows 8
}
let result = addWrong(3, 5);
console.log(result); // undefined!
✅ return (data comes back)
function addRight(a, b) {
  return a + b; // sends 8 back
}
let result = addRight(3, 5);
console.log(result * 2); // 16 ✅

🔭 What is Scope?

Scope determines where a variable is accessible. Variables declared inside a function are local — they only exist while the function is running, and cannot be accessed from outside it.

💡 Why is this good? Scope is a feature. It prevents different parts of your program from accidentally interfering with each other. Functions can have their own private variables without worrying about name clashes elsewhere.

🗺️ Global vs Local Scope — Visual

Global Scope
playerName= "Alice"
gameLevel= 5

↑ Accessible everywhere in your script

calculateBonus() — Local Scope
bonus= 5000

↑ Only accessible inside calculateBonus()

✅ Can read playerName from global scope

❌ Trying to access bonus outside the function → ReferenceError
