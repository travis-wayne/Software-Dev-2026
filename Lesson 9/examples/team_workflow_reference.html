Code Review Etiquette

Reviewing a teammate's Pull Request requires technical precision and human empathy.

✅ The Dos

  • 👍 Critique the code, not the person.
  • 📝 Explain *why* a change is needed:
    "Using flex-col here prevents the squishing bug on mobile."
  • 🔎 Check the exact scope. Did they only touch the Hero section like the ticket asked?
  • 🎉 Praise good code! "This CSS solution is really clever!"

❌ The Don'ts

  • 👎 "You did this wrong" or "This is bad."
  • 🛑 Merging a PR blindly without reading the 'Files Changed' tab.
  • 👻 Ignoring PRs. The longer a PR sits open, the higher the chance of merge conflicts.
  • 🔧 Scope creep: "While you're here, can you also fix the footer font?" (No! That is a separate ticket/branch!)
