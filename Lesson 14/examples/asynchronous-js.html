⏱️ The Blocking Problem

JavaScript is single-threaded. If a task (like a massive calculation or waiting for a server) blocks the main thread, the entire browser freezes. Users can't scroll, click, or type.

Try it: Click the "Run Blocking" button below. Notice how the console logs "Done" only after the loop finishes. In a real app, the UI would be dead during this time.

📦 Understanding Promises

A Promise is like a pager at a restaurant. You are promised food. While you wait, the promise is Pending. Eventually, it is either Fulfilled (success) or Rejected (failure).

🔗 Consuming a Promise: .then() / .catch() / .finally()

Once you have a Promise, you consume it with these chained methods. The key rule: each method returns a new Promise, so you chain them flat — no nesting.

MethodRuns whenReceives
.then(fn)Promise is FulfilledThe resolved value
.catch(fn)Promise is RejectedThe rejection reason / error
.finally(fn)Always — success or failureNothing (no argument)
💡 Tip: .finally() is perfect for cleanup — hiding a loading spinner regardless of whether the request succeeded or failed. 
fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(err => console.error('Failed:', err))
    .finally(() => hideLoadingSpinner());

✨ Async/Await: The Modern Standard

Promises are powerful but can get messy with .then() nesting. async/await allows you to write async code that looks like regular synchronous code.

Tip: Always use try...catch to handle errors when using await.

📡 Fetch API: Getting Real Data

The fetch() method makes an HTTP GET request and returns a Promise. It takes two awaits to get to the data.

⚠️ Critical Gotcha: fetch() only rejects on a complete network failure (no internet, bad DNS). If the server replies with a 404 or 500, the Promise still resolves. You MUST manually check response.ok to detect server errors.
StepCodeWhat it does
1await fetch(url)Gets the Response object (status + headers)
1bif (!response.ok) throw new Error(...)Manually catch 4xx / 5xx errors
2await response.json()Reads + parses the JSON body into a JS object
