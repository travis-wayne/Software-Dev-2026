🔗 Linking JavaScript to HTML

Just like CSS uses <link>, JavaScript uses the <script> tag. Place it at the bottom of <body> so the HTML loads first.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Page</title>
</head>
<body>
    <h1>Hello World</h1>

    <!-- JS goes at the BOTTOM of body -->
    <script src="script.js"></script>
</body>
</html>

⚔️ var vs let vs const

JavaScript has three keywords for declaring variables. In modern JS, you should only use two of them.

Keyword Re-assignable? Re-declarable? When to Use
const ❌ No ❌ No Default choice — value won't change
let ✅ Yes ❌ No Value will be updated later
var ✅ Yes ✅ Yes ❌ Never — legacy only 
// const — value is LOCKED
const courseName = "Software Dev 2026";
courseName = "Something else";  // ❌ TypeError!

// let — value CAN change
let score = 100;
score = 150;  // ✅ No problem

// var — allows accidental re-declaration (DANGEROUS)
var name = "Alice";
var name = "Bob";  // ⚠️ No error — silently overwrites!
⚠️ The Rule: Use const by default. Only use let when the value needs to change. Never use var in modern JavaScript.

🤔 undefined vs null

These two are easy to confuse:

  • undefined = "JavaScript says: you never gave this a value."
  • null = "The developer says: I intentionally set this to nothing."
let middleName;                // undefined — never assigned
let selectedItem = null;      // null — deliberately empty

⚠️ Why "5" + 3 = "53" but "5" - 3 = 2?
The + operator has two jobs: addition AND concatenation. If either side is a string, it concatenates. The -, *, / operators only do math, so they force the string to become a number.
